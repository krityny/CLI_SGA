                 $ cat output.txt
      output: I/O Operation Test
     (For Error): 
      Command:$ ls missing_file.txt 2> error.txt
              $ cat error.txt
      Output: ls: cannot access 'missing_file.txt': No such file or directory

  Explanation: Here for output redirection, the output of echo command is redirected to the output.txt file, whose content is then printed 
using the cat command followed by file name.
   For error redirection, the error message from the ls missing-file.txt is redirected to error.txt file instead of being printed on the 
screen. The contents from the error.txt file is then printed with the cat command.

5. How Linux Manages file IO:
Linux uses file descriptors to manage input and output streams. The standard descriptors are 0 for input, 1 for normal output and 2 for error output. Accessing files for using read write or close commands is handles by the linux kernel.
We can also redirect input and error using FD1 and FD2 respectively. Linux treats all processes as open files. If an application exceeds its open file limit, it may cause logging failures.
